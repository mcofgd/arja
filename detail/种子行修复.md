## 🎯 核心问题解决

**根本原因**: 
- `seedLineGenerated=false` → `seedLines=null`
- InitASTVisitor.java:91 的判断 `if (seedLines == null || ...)` 永远为 true
- 导致所有语句（5000+）都被加入 ingredient 池
- 过滤规则失效，修复能力归零

**解决方案**:
- 使用 `faultyLines.keySet()` 作为种子行
- 既解决 Java 11 兼容性，又保持精确性
- 恢复 ARJA 的精确搜索能力
